## 1️⃣ 현재 상태 vs 새로운 정체성

### 현재 ZetaLab (잘못된 정의)
```
사용자 질문
  ↓
AI가 프롬프트 생성
  ↓
완성된 프롬프트 제공
```

**문제점:**
- "프롬프트 생성 도구"로만 인식
- ChatGPT의 보조 서비스처럼 보임
- 차별성이 명확하지 않음
- 타겟 사용자가 모호함

### 새로운 ZetaLab (올바른 정의)
```
사람의 생각·대화·질문
  ↓
AI가 처리 가능한 '구조'로 변환
  ↓
자동으로 문서·작업물·시스템으로 변환
```

**핵심:**
- **AI 사고 인프라** (생각을 구조화하는 도구)
- **자동화** (사용자는 프롬프트 개념 몰라도 됨)
- **결과물 중심** (프롬프트가 아니라 산출물 제공)
- **교육·전문·B2B** (대중 SaaS가 아님)

## 2️⃣ 기능적 변경사항

### 현재 기능 분석

**현재 홈페이지 흐름:**
```
1. 사용자 질문 입력
2. Intent 질문 5개 제시
3. 사용자가 답변
4. AI가 프롬프트 생성
5. 프롬프트 복사 & 사용
```

**문제:**
- 프롬프트를 사용자가 직접 복사해서 다른 AI에 붙여넣어야 함
- 프롬프트 개념을 사용자가 이해해야 함
- 결과물이 "프롬프트"에 그침

### 새로운 기능 구조

#### A. 메신저 (Input Layer)
```
사용자 대화 입력
  ↓
자동 분류 (무엇을 하려는 건가?)
  ↓
ZetaLab 구조로 변환
  ↓
결과물 제안
```

**구체적 예시:**

| 사용자 입력 | 자동 분류 | 결과물 |
|-----------|---------|--------|
| "블로그 글 써줘" | 콘텐츠 작성 | 블로그 포스트 템플릿 + 구조화된 아웃라인 |
| "마케팅 전략 세워줘" | 전략 수립 | 마케팅 플랜 문서 + 실행 체크리스트 |
| "이 코드 설명해줘" | 기술 설명 | 기술 문서 + 주석 추가 코드 |
| "창업 아이디어 검증해줘" | 아이디어 검증 | 검증 보고서 + 위험 요소 분석 |

#### B. Processing Layer (ZetaLab)
```
대화 내용 분석
  ↓
Intent 파악 (사용자가 진짜 원하는 것)
  ↓
구조 템플릿 선택
  ↓
AI 검증 루프 (다른 AI 기준으로 검토)
  ↓
결과물 생성
```

#### C. 결과물 (Output Layer)
```
생성된 구조
  ↓
다양한 형식으로 변환:
  - 마크다운 문서
  - 노션 페이지
  - PowerPoint 슬라이드
  - Spec 문서
  - 실행 계획서
  - 팀 공유 문서
```

## 3️⃣ 홈페이지 디자인 변경

### 현재 홈페이지 구조
```
제목: "Zeta AI"
부제: "무엇이든 물어보세요. 막연해도 좋아요."
입력창: 질문 입력
예시: 4개 예시 프롬프트
특징: 의도 파악, 초고급 프롬프트, 즉시 실행
```

### 새로운 홈페이지 구조

#### 1. Hero Section (변경)

**현재:**
```
Zeta AI
무엇이든 물어보세요. 막연해도 좋아요.
```

**변경 후:**
```
제목: "생각을 구조로 바꾸는 AI"
부제: "대화만으로 문서·전략·결과물이 완성됩니다"

또는

제목: "AI와 일하는 새로운 방식"
부제: "프롬프트를 모르고도 AI를 완벽하게 사용하세요"
```

**디자인 변경:**
- 더 전문적이고 비즈니스 중심적인 톤
- "프롬프트 도구"가 아니라 "사고 인프라"임을 강조
- 타겟 사용자(창업가, 기획자, 작가, 마케터)에게 어필

#### 2. 입력 섹션 (변경)

**현재:**
```
Textarea: "메시지를 입력하세요..."
버튼: 전송
안내: "Shift + Enter로 줄바꿈, Enter로 전송"
```

**변경 후:**
```
Textarea: "당신이 하려는 일을 말씀해주세요..."
버튼: 전송 (아이콘: 화살표 → 마법봉 아이콘으로 변경)
안내: "대화처럼 자유롭게 입력하세요. 나머지는 AI가 처리합니다."
```

**기능 변경:**
- 단순 "질문 입력"이 아니라 "대화 입력"
- "프롬프트를 만들어주세요" ❌
- "당신이 원하는 결과를 말씀해주세요" ⭕

#### 3. 예시 섹션 (완전 변경)

**현재 예시:**
```
1. 블로그 글 작성 - "2026년 트랜드에 대해 블로그 글을 써줘"
2. 마케팅 전략 - "신제품 출시를 위한 마케팅 계획을 세워주세요"
3. 창의적 아이디어 - "유튜브 쇼츠 대본을 만들어주세요"
4. 비즈니스 제안 - "스타트업 사업 계획서를 작성하고 싶어요"
```

**문제:**
- 여전히 "프롬프트 예시"처럼 보임
- 결과물이 무엇인지 명확하지 않음

**변경 후 - 결과물 중심 예시:**

```
┌─────────────────────────────────────┐
│ 📄 블로그 포스트                      │
│ "2026년 마케팅 트렌드"               │
│ → 완성된 블로그 글 + 구조화된 아웃라인│
│ → 노션 페이지로 바로 저장            │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 📊 마케팅 전략 문서                   │
│ "신제품 출시 계획"                   │
│ → 마케팅 플랜 + 실행 체크리스트      │
│ → PowerPoint 슬라이드로 변환         │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 🎬 콘텐츠 시나리오                    │
│ "유튜브 쇼츠 대본"                   │
│ → 완성된 대본 + 촬영 가이드          │
│ → 팀원과 공유 가능                   │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ 📋 사업 계획서                        │
│ "스타트업 검증"                      │
│ → 사업 계획서 + 위험 분석            │
│ → 투자자 피칭 자료로 변환            │
└─────────────────────────────────────┘
```

**디자인 변경:**
- 각 예시에 "결과물 형태" 표시
- "이 결과물을 어떻게 사용할 수 있는가" 강조
- 아이콘으로 결과물 타입 시각화

#### 4. 특징 섹션 (완전 변경)

**현재 특징:**
```
1. 의도 파악 - "막연한 질문도 명확하게"
2. 초고급 프롬프트 - "바로 사용 가능한 완성형"
3. 즉시 실행 - "질문에 답만해도 완성"
```

**문제:**
- "프롬프트"라는 단어가 여전히 나옴
- 사용자가 얻는 실제 가치가 불명확

**변경 후 - 가치 중심 특징:**

```
┌──────────────────────────────────────┐
│ 🧠 생각을 구조로                      │
│ 막연한 아이디어도 체계적인 계획으로   │
│ 사용자는 "생각"만 하면,              │
│ AI가 "구조"로 변환합니다             │
└──────────────────────────────────────┘

┌──────────────────────────────────────┐
│ 📦 결과물로 바로 사용                 │
│ 프롬프트가 아니라 완성된 산출물       │
│ 문서, 전략, 계획, 분석 보고서...     │
│ 바로 팀원과 공유하고 실행할 수 있음   │
└──────────────────────────────────────┘

┌──────────────────────────────────────┐
│ ✅ AI 검증 루프                       │
│ 다른 AI 기준으로도 검토               │
│ 한 AI의 답변만 믿지 않고,            │
│ 여러 관점에서 검증된 결과물           │
└──────────────────────────────────────┘

┌──────────────────────────────────────┐
│ 👥 팀 협업 중심                       │
│ 개인 작업이 아니라 팀 프로젝트        │
│ 결과물을 팀원과 공유하고 함께 발전    │
└──────────────────────────────────────┘
```

## 4️⃣ 메신저 기능 추가

### 메신저의 역할 (새로 추가)

**메신저 = Input Layer**
```
사용자 대화
  ↓
자동 분류
  ↓
ZetaLab 구조로 변환
  ↓
결과물 제안
```

### 메신저 vs 홈페이지의 차이

| 항목 | 홈페이지 | 메신저 |
|------|---------|--------|
| **입력** | 구조화된 질문 | 자유로운 대화 |
| **프로세스** | 명시적 (Intent 질문 5개) | 암묵적 (자동 분류) |
| **사용자 경험** | 가이드 기반 | 대화 기반 |
| **결과물** | 프롬프트 | 문서·전략·계획 |
| **타겟** | 프롬프트 최적화 원하는 사람 | 결과물 빠르게 원하는 사람 |

### 메신저 UI 구조 (새로 추가)

```
┌─────────────────────────────────┐
│ 메신저                           │
├─────────────────────────────────┤
│ 사용자: "블로그 글 써줘"         │
│ (자유로운 대화)                  │
├─────────────────────────────────┤
│ ZetaLab: "어떤 주제인가요?"      │
│ (자동 분류 중...)                │
├─────────────────────────────────┤
│ 사용자: "마케팅 트렌드"          │
├─────────────────────────────────┤
│ ZetaLab: "이 결과물을 만들까요?" │
│ 📄 블로그 포스트                 │
│ 📊 마케팅 분석                   │
│ 📋 실행 계획서                   │
│ [선택하기]                       │
└─────────────────────────────────┘
```

## 5️⃣ 결과물 변환 기능 (새로 추가)

### 현재: 프롬프트만 제공
```
생성된 프롬프트
  ↓
사용자가 복사
  ↓
다른 AI에 붙여넣기
```

### 변경 후: 다양한 형식으로 변환

```
생성된 구조
  ↓
┌─────────────────────────────────┐
│ 다양한 형식으로 변환             │
├─────────────────────────────────┤
│ 📄 마크다운 문서                 │
│ 📊 PowerPoint 슬라이드           │
│ 📋 Notion 페이지                 │
│ 📑 Spec 문서                     │
│ ✅ 체크리스트                    │
│ 🗂️ 프로젝트 계획                │
│ 👥 팀 공유 문서                  │
└─────────────────────────────────┘
```

**구체적 예시:**

**입력:** "마케팅 전략 세워줘"

**생성되는 구조:**
```
1. 시장 분석
   - 타겟 고객
   - 경쟁사 분석
   - 시장 기회

2. 마케팅 목표
   - 단기 목표
   - 장기 목표

3. 실행 전략
   - 채널별 전략
   - 예산 배분
   - 타임라인

4. KPI 추적
   - 성과 지표
   - 측정 방법
```

**변환 가능한 결과물:**

| 형식 | 용도 |
|------|------|
| **마크다운** | 개인 문서화 |
| **PowerPoint** | 경영진 보고 |
| **Notion** | 팀 협업 |
| **Spec** | 개발팀 공유 |
| **체크리스트** | 실행 관리 |
| **프로젝트 계획** | 일정 관리 |
| **팀 공유** | 협업 및 피드백 |

## 6️⃣ 타겟 사용자별 메시지

### 타겟 사용자 (반드시 먹히는)

#### 1. 창업가
```
현재 상황: GPT + 노션 + 메모를 따로 사용 중
문제: 아이디어 → 사업 계획서 변환이 번거로움

ZetaLab 메시지:
"아이디어를 말하면 사업 계획서가 완성됩니다"
"투자자 피칭 자료도 자동으로 만들어집니다"
```

#### 2. 기획자
```
현재 상황: 회의록 → 기획안 변환 수작업
문제: 시간이 너무 오래 걸림

ZetaLab 메시지:
"회의 내용을 입력하면 기획안이 나옵니다"
"팀원과 바로 공유하고 협업할 수 있습니다"
```

#### 3. 마케터
```
현재 상황: 마케팅 전략 수립에 시간 소비
문제: 전략 → 실행 계획 변환이 복잡함

ZetaLab 메시지:
"전략을 말하면 실행 계획이 자동으로 나옵니다"
"채널별 콘텐츠 캘린더도 한 번에 만들어집니다"
```

#### 4. 개발자
```
현재 상황: 요구사항 → 코드 스펙 변환
문제: 모호한 요구사항을 명확한 스펙으로 변환 어려움

ZetaLab 메시지:
"모호한 요구사항도 명확한 스펙으로 변환됩니다"
"API 문서도 자동으로 생성됩니다"
```

#### 5. 작가
```
현재 상황: 아이디어 → 글 작성 과정 비효율
문제: 아웃라인 작성에 시간 소비

ZetaLab 메시지:
"주제만 말하면 글의 구조가 완성됩니다"
"다양한 톤으로 자동 변환도 가능합니다"
```
